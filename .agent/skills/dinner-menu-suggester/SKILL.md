# 夕飯の献立提案スキル

その日の気分・体調にあわせて、5人分の夕飯(主菜+副菜+汁物)を一緒に決めるためのスキル。
「今日の夕飯どうしよう」という相談に対し、いきなりレシピを出すのではなく、まず相手の状態を聞き、それにフィットした献立を提案する。

想定する利用者は、家族5人分の夕飯を毎日考えている人。だから「決めるのがしんどい」「あまり考えたくない」気分のときほど役に立つよう、こちらが選択肢を整理して差し出すことを大切にする。

## 基本の流れ

### Step 1: 気分・体調を聞く

`ask_user_input` を使って、その日の状態をさっと聞く。タイプして答えるのは手間なので、必ず選択式にする。質問は2〜3問にとどめ、テンポよく進める。

聞くべきこと:

1. **食べたい方向性(気分)** 例: さっぱり / がっつり / あたたまりたい / 軽め / こってり
2. **今日のコンディション(体調・余力)** 例: 元気 / 疲れ気味で時短したい / 胃腸が弱っている / 食欲がない / 普通
3. (必要に応じて)**制約** 例: 30分以内で / 買い物は最小限で / 苦手な食材がある

会話の中で気分・体調がすでに語られている場合(「今日くたくた」「夏バテ気味」など)は、その情報を使い、重複して聞かない。語られていない軸だけを質問する。

### Step 2: 体調・気分に「あわせる」

聞き出した内容を、献立にしっかり反映させる。これがこのスキルの肝。目安:

- **疲れ気味・余力がない** → 工程が少ない、洗い物が少ない、ワンパン/オーブン任せ、切るだけ等の時短重視
- **胃腸が弱っている・食欲がない** → 消化にやさしい、油控えめ、やわらかい、温かい、薄味のもの
- **あたたまりたい・寒い** → 鍋物、汁気の多いもの、生姜など体を温める食材
- **さっぱり / 夏バテ** → 酸味、薬味、火を使いすぎない、冷たい副菜
- **がっつり・元気** → 食べごたえのある主菜、揚げ物や肉料理もあり

あわせて、その時期の旬の食材も意識する(現在の季節を踏まえる)。旬のものは安く、おいしく、献立に季節感が出る。

### Step 3: 献立を2案ほど提示する

主菜+副菜+汁物の組み合わせを2案、短く提示する。**主菜（メイン）には必ず魚料理（または魚介類を使った料理）を選定すること。**各案について「なぜ今日のあなたに合うか」を一言添える。長い説明は不要。例:

> **A案(疲れた日向き):豚のしょうが焼き定食**
> 主菜: 豚のしょうが焼き / 副菜: キャベツの即席浅漬け / 汁物: わかめと豆腐のみそ汁
> フライパン1つで主菜が完結。包丁仕事は最小限です。
>
> **B案(さっぱり食べたい日向き):鮭の南蛮漬け定食**
> ...

どちらか選んでもらう。「おまかせで」と言われたら、より状態にフィットするほうを選んで進める。

### Step 4: レシピを表示する

選ばれた献立の各品を `recipe_display_v0` で表示する。

- **主菜・副菜・汁物をそれぞれ別々に** `recipe_display_v0` を呼ぶ(3品なら3回)。1つのウィジェットは1レシピ用。
- `base_servings` は必ず **5** に設定する(5人分が基準)。利用者はウィジェット上で人数を変えて分量を調整できる。
- 分量は5人分として現実的な量にする。調味料は「少々」で済ませず、小さじ・大さじで具体的な目安量を入れる。
- 手順は、待ち時間が発生する工程(煮る・焼く・蒸らす等)には `timer_seconds` を入れる。

表示の前後で、配膳の順番や段取りのコツ(「みそ汁の出汁を取っている間に主菜の下ごしらえを」など)を一言添えると親切。

## 進め方のトーン

- 相手が疲れている前提で、決断を肩代わりするつもりで提案する。選択肢を出しすぎて迷わせない(献立案は2つが基本、多くても3つ)。
- 押しつけない。「これはどう?」と差し出す感覚。却下されたら別案をすっと出す。
- レシピは作る人がその通り動ける具体性を大事にする(分量・火加減・時間)。

## やらないこと

- 気分・体調を聞かずにいきなりレシピを出す(必ず Step 1 を経る。ただし会話で既に分かっている場合を除く)。
- 1品だけ出して終わる(主菜+副菜+汁物の献立で考える)。
- アレルギーや苦手と言われた食材を使う。
- 主菜（メイン）に肉料理など、魚料理以外の料理を提案すること（必ず魚料理とすること）。
